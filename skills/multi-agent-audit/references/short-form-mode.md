# Short-form executive-summary mode (v1.3+)

The full audit report (12 sections, ~25 KB markdown) is the right output for a maintainer who wants the evidence chain. For an exec briefing or a Slack post, that's too much. **Short-form mode** produces a single-page output (markdown or HTML) optimized for "30-second read":

- Verdict pattern + one-line summary
- Headline metrics (intervention tax, autonomy split, operational fidelity)
- Top 3 ranked opportunities
- Pointer to the full report

Always produced **alongside** the full report — short-form is additive, not replacement.

## Markdown short-form

Generated by `scripts/render_short.py`. Target: ≤1 KB.

Output:

```markdown
# Multi-agent audit — {{PROJECT}} — {{DATE}}

**Verdict:** {{PATTERN}}
*{{ONE_LINE_SUMMARY}}*

| Metric | Value |
|---|---|
| Intervention tax | {{INTERVENTION_TAX}} |
| Autonomy split | {{AUTONOMY_SPLIT}} |
| Operational fidelity | {{FIDELITY}} |

## Top opportunities

1. **{{OPP1_WHAT}}** — {{OPP1_WHY}} *(score {{OPP1_SCORE}}/9)*
2. **{{OPP2_WHAT}}** — {{OPP2_WHY}} *(score {{OPP2_SCORE}}/9)*
3. **{{OPP3_WHAT}}** — {{OPP3_WHY}} *(score {{OPP3_SCORE}}/9)*

Full report: [{{FULL_REPORT_PATH}}]({{FULL_REPORT_PATH}})
Snapshot: `{{SNAPSHOT_PATH}}`
```

## HTML short-form

Same content, rendered as a self-contained ~5 KB HTML page (no Chart.js, just a styled card layout). Suitable for inline-render in email digests or pinned at the top of a Slack channel.

Use this when the audience won't open a file: the HTML displays directly in many email clients.

## Invocation

```bash
# Markdown short-form (default)
python3 scripts/render_short.py --snapshot <snapshot.json> --output <short.md>

# HTML short-form
python3 scripts/render_short.py --snapshot <snapshot.json> --output <short.html> --format html

# Both, alongside the full report
python3 scripts/render_short.py --snapshot S.json --output short.md
python3 scripts/render_short.py --snapshot S.json --output short.html --format html
python3 scripts/render_report.py  --snapshot S.json --output full.html
```

The auditor invokes both in Step 4 by default; the human picks which to share with whom.

## Addenda behavior

Like the full report, short-form **applies `addenda[*].revised_values`** before rendering. The body values are provenance; the surfaced numbers are what the audit's latest understanding says.

## When short-form is the wrong format

- When the reader needs the methodology / caveats / sampling notes to calibrate confidence.
- When deltas between snapshots matter (full report has trend; short-form skips it).
- When per-persona breakdown is the point.

For those, send the full report.

## Future extensions

v1.4 candidates noted for short-form:

- A "since last audit" delta line in headline metrics (when ≥2 snapshots exist) — *"Intervention tax 0.62 (↓0.19 since 2026-05-12)"*.
- A 1-line auditor-independence note when applicable.
- Embed as a comment-able PR description so audit findings can be discussed inline on a "review the audit" PR.
